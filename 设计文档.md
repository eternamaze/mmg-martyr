# Martyr 最终宪章：主权临界区与生命周期禁锢协议 (V3.Final)
## 核心法则：所有权锁死、双向审计、禁锢访问、逾期殉葬

---

### 1. 权力根基：资源禁锢 (Resource Confinement)
`Martyr` 的存在是为了终结“所有权模糊”的时代。资源不属于任何使用者，它仅在执行期间被**“禁锢”**在临界区内。

* **禁止外传 (Non-Escapability)**：利用 Rust 的高阶闭包与匿名生命周期（Higher-Rank Trait Bounds 思想），确保资源引用 `&T` 的存续期严格等于闭包执行期。
* **物理隔离**：开发者永远无法接触资源的真实句柄，只能接触到被哨兵严格监控的“镜像借用”。
* **结果**：离开哨兵大门的那一刻，访客在物理上**不可能**持有任何关于资源的残留信息。



---

### 2. 双向哨兵机制 (The Bi-directional Sentry)
哨兵是主权中心伸向业务层的触角，负责全生命周期的实名审计。

* **进入感知 (Check-in)**：
    * **身份绑定**：记录 `ThreadId/TaskHandle` 与业务标识。
    * **处决契约**：一旦进入，即代表签署了“殉葬协议”。主权中心获得在必要时终止该上下文的合法权能。
* **离场注销 (Check-out)**：
    * **状态清白证明**：基于 RAII 的 `Drop` 机制。只有当你彻底丢弃资源引用、退出闭包后，注销才会发生。
    * **脱离处决名单**：注销成功意味着主权中心确认你已“净身出户”，不再处于核打击范围。

### 3. 实现原理：生命周期时空锁定

```rust
impl<'a, T, D: Discipline> Sentry<'a, T, D> {
    /// 唯一的访问门户：execute 模式强制实现了“带不走”与“必须打卡”
    pub fn execute<F, R>(&self, action: &'static str, f: F) -> R 
    where 
        for<'b> F: FnOnce(&'b T) -> R // 匿名生命周期禁锢：引用无法逃离闭包
    {
        // 1. 进入登记：建立处决连结
        let _guard = self.registry.check_in(action);

        // 2. 主权检查：关灯后禁止进入
        if self.registry.is_killed() {
            panic!("🔥 [Martyr] 区域已封锁，强行闯入者死！");
        }

        // 3. 业务执行：资源在禁锢区内流动
        let result = f(self.inner);

        // 4. 离开注销：RAII 自动触发。
        // 若此处因 shutdown 被阻断，则 _guard 永不释放，主权中心立即执行处决。
        result
    }
}
```

### 4. 最终审判：清场与殉葬逻辑

当主权中心发起 `force_shutdown` 时，它执行的是一场基于考勤表的精准清洗：

1. **瞬时熔断**：原子开关翻转，拒绝一切新的 `Check-in`。
2. **存量审计**：实时扫描 `VisitorList`。
3. **逻辑判定**：
    * **名单为空**：证明所有访客已完成 `Check-out`。基于 Rust 编译器的保证，此时**全系统没有任何角落持有资源引用**。物理析构安全执行。
    * **名单不为空**：证明存在滞留者（无论是因为代码臃肿还是恶意泄露）。
4. **同步处决**：
    * 根据 `Discipline` 策略，对名单上的人员发起精准打击（Abort 协程或毒化线程）。
    * **殉葬哲学**：如果你宣称你没离开，那你就必须死在里面，以确保主权中心能安全收回生命周期。
